=Ö Key Concepts Explained

Abstract Classes

Abstract classes serve as templates. They can't be instantiated directly but provide a base for other classes. The Person class is abstract because people in our system are either Developers or Designers, never just "Person".

Inheritance

Both Developer and Designer "inherit" from Person, meaning they get all Person's properties and methods while adding their own specific features.

Access Modifiers

  • public - Accessible from anywhere (default)
  • private - Only accessible within the class
  • protected - Accessible in class and subclasses

Generics

Generics let you write flexible, reusable code. The <T extends Person> syntax means "this works with Person or any class that extends Person".
